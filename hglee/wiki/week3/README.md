# 1. 트리
## 트리 개념 정리
- 데이터를 부모-자식의 계층 구조로 표현하여 저장한 자료구조
- 트리를 이루는 각각의 데이터를 노드라 한다.
- 최상위 노드(루트 노드)를 시작으로 1개이상의 자식 노드를 가지며 특정 노드부터 그 하위 형태 또한 트리 구조이다.
- 노드가 N개인 트리는 항상 N-1개의 간선을 가진다. 
- 루트 노드에서 특정 노드로 가는 경로는 유일하다.

## 트리 용어 정리
- 루트 노드 : 트리의 최상위 노드
- 간선 : 노드와 노드를 연결하는 선
- 깊이 : 루트 노드에서 특정 노드까지 도달하는데 거치는 간선의 갯수. 루트 노드의 깊이는 0
- 레벨 : 해당 노드의 깊이 + 1. 루트 노드의 레벨은 1
- 부모 노드 : 부모 자식 관계에서 상위측의 노드
- 자식 노드 : 부모 자식 관계에서 하위측의 노드
- 형제 노드 : 부모가 동일한 노드
- 조상 노드 : 특정 노드로 부터 루트 노드까지 거치는 모든 부모 노드들의 모음
- 후손 노드 : 특정 노드를 루트로 삼았을때 형성되는 서브트리에서 자신 제외한 모든 노드
- 노드의 차수 : 노드의 자식 수
- 트리의 차수 : 해당 트리의 노드들의 차수중 최대값
- 단말 노드 : 자식이 없는 노드

## 이진 트리
모든 노드의 최대 차수를 최대 2로 제한한 트리 
- 자식노드가 아예 없거나, 왼쪽에만 있거나, 오른쪽에만 있거나 왼쪽 오른쪽 모두 있을수있음

## 완전 이진 트리
이진 트리이면서 노드가 왼쪽부터 채워져있는 트리
- 자식노드가 없거나, 왼쪽에만 있거나, 왼쪽 오른쪽 모두 있을수있음 (오른쪽에만 있을수는 없음)
- 단, 왼쪽에만 있는 경우는 마지막 레벨에서만 허용됨

## 포화 이진 트리
이진 트리이면서 모든 노드의 자식이 없거나 2개인 트리
- 마지막 레벨의 노드는 자식이 없음
- 마지막 레벨의 노드를 제외한 모든 노드는 자식이 왼쪽 오른쪽 둘다 있음

# 2. 힙
힙은 완전 이진 트리의 구조에 부모 - 자식 관계에 우선순위 개념을 추가한 자료 구조이다.
즉 부모 노드는 항상 자식 노드보다 우선순위가 앞선다는 조건으로 완전 이진 트리 형태로 채워나가는 것이다.
힙의 포인트는 우선순위가 가장 높은 값을 빠르게 찾아낸다는 것이고, 우선순위 큐 구현의 중심이 됨 
대표적으로 최대 힙, 최소 힙이 있다.

## 힙의 주요 특징
- 당연하지만, 루트 노드는 힙 내(해당 완전 이진 트리)에서 우선순위가 가장 높은 자료이다.
- O(1)의 복잡도로 우선순위가 가장 높은 건을 뽑아낼 수 있다.
- 노드를 추가할 때 부모 - 자식만 비교하면 되기때문에 최대 트리의 깊이만큼만 비교를 하면되서 O(logN)의 복잡도로 추가할 수 있다.
- 부모 - 자식 간의 우선순위에만 중점을 두기 때문에 형제간 우선순위는 고려되지 않는다.
    - 정렬이 완벽하게 되어있지 않다는 뜻. 이를 반 정렬 상태 혹은 느슨한 정렬 상태라고 함

## 최소 힙
- 부모 노드의 값이 자식 노드의 값보다 항상 작은 힙
- 루트 노드는 해당 힙의 최솟값

## 최대 힙
- 부모 노드의 값이 자식 노드의 값보다 항상 큰 힙
- 루트 노드는 해당 힙의 최댓값
